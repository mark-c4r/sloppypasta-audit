## Audit Scope

**Surfaces assessed**: fountain.fm landing page (direct fetch); Substack blog posts (release notes examined); Google Play data safety label (20 permissions, 18 libraries, third-party data sharing confirmed); Twitter/X profile; Stacker News presence; App Store listing; GitHub org fountain-fm (5 repos — all utility libraries, main app absent).
**Surfaces via proxy**: Native app behavior inferred from store listings, user reviews (battery drain, earning issues), and privacy policy — apps not installed. Web player limitations from user reports. Custodial mechanics from privacy policy and published withdrawal threshold.
**Surfaces not assessed**: Native iOS/Android apps (not installed); logged-in web experience; earning mechanics; NWC flow; podcaster dashboard; live Lightning payments; account deletion. Privacy, terms, about pages all returned 404.
**Limitations**: Multiple key pages returned 404. All [if:mobile] checks proxy-derived from store listings and user reviews.

---

## Overall: 54% -- Grade D

Fountain occupies a contradictory position: it is built on genuinely open protocols (RSS, Lightning, Nostr) but wraps them in a closed-source, custodial, app-store-dependent package. The writing and slop detection scores are decent -- the blog is clearly human-written with minimal AI tells, and the copy avoids the worst marketing bloat. But the sovereignty score (39%) drags the overall down hard. A custodial wallet with a 100,000 sat minimum withdrawal, no OPML export, closed-source main app, no F-Droid distribution, and explicitly ignored DNT signals undercut the "open" narrative. The product has real craft and a real community, but the gap between its protocol-level aspirations and its platform-level execution is the defining tension.

## Priority Improvements

1. **Fix broken legal pages (4.2, currently 45 -> target 70, high impact)**: Privacy policy, terms of service, and about pages all return 404 on the main website. These are table-stakes pages that signal basic product care and legal compliance. Fixing them would improve Product Craft and multiple transparency checks simultaneously.

2. **Ship OPML export and clear account deletion (6.2 + 5.7, currently 30/40 -> target 60/65, high impact)**: No visible data export mechanism for podcast subscriptions. Account deletion path is unclear ("contact support"). Implementing one-click OPML export and self-service account deletion would significantly improve Exit Cost and Cancel/Leave scores. This is the single highest-leverage change for sovereignty.

3. **Move wallet from custodial to non-custodial default (6.13 + 6.11, currently 25/40 -> target 75/75, transformative)**: The custodial wallet with 100,000 sat minimum withdrawal contradicts the Bitcoin sovereignty narrative. Completing the NWC (Nostr Wallet Connect) integration and making external wallet connection the default -- not an opt-in extra -- would transform the sovereignty category. This is the change most aligned with Fountain's stated values.

4. **Open source the main app (6.5 + 6.7, currently 25/25 -> target 75/75, high impact)**: The GitHub org has only utility libraries. Open-sourcing the main app (even under AGPL) would dramatically improve auditability and source code scores. For a product built on open protocols, closed-source implementation is a credibility gap.

5. **Honor DNT and reduce third-party data sharing (5.5e + 5.5c, currently 5/25 -> target 75/75, moderate impact)**: Explicitly ignoring DNT while sharing data with advertising partners is the worst single finding in this audit. Honoring DNT and switching to privacy-respecting analytics (self-hosted Plausible/Umami) would lift the entire 5.5 composite from 30% toward 60%+.
